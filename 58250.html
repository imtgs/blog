用Termux+Github+Hexo搭建个人博客

更新于2022-05-21|教程干货Hexo
|字数总计:1.5k|阅读时长:5分钟|评论数:

由于我的电脑被我妈带走了，所以我也只能在手机上玩玩Termux。最近花了几天时间把博客框架换成了Hexo，心血来潮写了这篇教程

本教程适用于Android端，理论上也适用于Linux端（毕竟都是Linux环境）

先来介绍一下本教程用到的东西

Termux

Termux是一款Android终端模拟器和Linux环境应用程序，无需root或设置即可直接运行。自动安装最小的基本系统，使用APT包管理器可以使用其他包。

这是本次教程所用到的软件

相关文档

https://f-droid.org/packages/com.termux/

博主不推荐使用其它下载方式，F-Droid下载的版本较新

遇到问题？

第一次启动Termux的时候需要从远程服务器加载数据，你可能会遇到一个弹窗

解决方法：

  1. 开飞机（懂得都懂），成功率高
  2. 切换到运营商流量，有概率成功

Github Pages

您可以使用 GitHub Pages 来展示一些开源项目、主持博客甚或分享您的简历。

简单来说，Github Pages 就是一个免费的服务器，我们的博客部署在这里就行

使用Github Pages需要Github账号，如果你连这都没有，就去问问百度吧

Hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

它是一个静态博客生成器，关于这东西的用法我就不多说了，本教程主题是使用Termux搭建博客，相关教程在下面

文档

https://hexo.io/zh-cn/docs/

配置博客

先说简单点的，博客根目录里_config.yml文件就是你博客的配置文件，打开它

找到对应的参数并更改，以下是我总结的一个表格

参数 解释
title 网站标题
subtitle 网站副标题，有的主题要在主题配置文件里配置
description 网站描述
keywords 网站的关键词，多个关键词就使用,隔开
author 填你自己的名字，用于在一些地方显示
language 网站使用的语言，不同的主题可能需要设置成不同的值，常见的有zh-Hans和zh-CN
timezone 网站时区，填Asia/Shanghai即可

一般就填上面的就可以了，注意冒号和逗号后面都要空一格。保存后在本地运行看效果，然后部署到Github
